## 개요

- 머신러닝이란 데이터로부터 스스로 규칙을 학습하여 예측·분류하는 알고리즘의 총칭임
- 명시적인 규칙을 코딩하는 대신, 데이터 안의 패턴을 자동으로 찾아냄

---

## 학습 유형

머신러닝은 정답 데이터(레이블)의 유무에 따라 세 가지로 나뉨

| 구분 | 정의 | 대표 알고리즘 |
|------|------|------|
| **지도학습** | 입력·정답 쌍으로 학습 | 선형회귀, SVM, 결정 트리 |
| **비지도학습** | 정답 없이 데이터 구조 파악 | K-Means, PCA, DBSCAN |
| **강화학습** | 환경과 상호작용하며 보상 최대화 | Q-Learning, PPO |

![머신러닝 학습 유형](/assets/img/ai-algorithms/ml-types.svg){: .normal}

---

## 학습 흐름

- 데이터 수집 → 전처리(결측치·스케일링) → 모델 선택 → 학습 → 평가 → 예측 순서로 진행됨
- 학습 데이터(train)로 모델을 훈련하고, 테스트 데이터(test)로 성능을 검증함
- 모델이 훈련 데이터에 과하게 맞춰지면 **과대적합(Overfitting)**, 너무 단순하면 **과소적합(Underfitting)**

---

## 주요 용어

- **특성(Feature)** : 예측에 사용하는 입력 변수 (예: 키, 나이)
- **레이블(Label)** : 예측 대상 값 (예: 집 가격, 클래스 번호)
- **하이퍼파라미터** : 학습 전 사람이 직접 설정하는 값 (학습률, 트리 깊이 등)
- **교차 검증(Cross-Validation)** : 데이터를 여러 번 분할해 성능을 안정적으로 측정하는 방법
- **스케일링** : 특성들의 값 범위를 맞춰주는 전처리 (StandardScaler, MinMaxScaler)

---

## 장단점

**장점**
- 복잡한 패턴을 데이터에서 자동으로 학습함
- 다양한 도메인(이미지, 텍스트, 수치 데이터)에 적용 가능함

**단점**
- 데이터 품질과 양에 크게 의존함
- 모델이 복잡해질수록 해석이 어려워짐 (블랙박스 문제)
