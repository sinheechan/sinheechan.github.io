## 이 가이드에 대하여

- 파이썬으로 다룰 수 있는 **모든 데이터 타입**을, 쉬운 설명 → 실제 메모리 크기 → 언제 쓰는지 순서로 정리한 가이드임
- 아래 메모리 수치는 전부 **CPython 3.12.3 / 64bit** 환경에서 `sys.getsizeof()` 로 직접 측정한 값임
- 읽는 순서: **바이트 개념 이해 → 숫자 → 묶음 → 글자/이진 → 이름표 → 날짜 → 대용량/표** 순으로 따라오면 됨

---

## 먼저 알아야 할 것: 컴퓨터는 모든 걸 '바이트'로 본다

- 컴퓨터의 메모리 안에는 사실 **0과 1(비트)** 밖에 없음
- 비트 8개를 묶은 것이 **1바이트(byte)** 이고, 컴퓨터는 어떤 데이터든 결국 **바이트의 나열**로만 저장함
- 숫자 `65`, 문자 `'A'`, 색상, 소리 — 저장된 실제 모습은 전부 똑같은 바이트 `01000001` 임

> **데이터 타입 = "이 바이트들을 어떻게 읽어야 하는지 알려주는 규칙"**
> 같은 바이트라도 "정수로 읽어라" 하면 숫자 65, "문자로 읽어라" 하면 글자 'A'가 됨

- 즉 데이터 타입은 데이터 자체가 아니라 **해석 방법**임 → 그래서 타입마다 "몇 바이트를 쓰는가"가 달라짐
- 이 가이드에서 크기를 계속 이야기하는 이유가 여기 있음: 결국 모든 데이터는 바이트로 환산되어 메모리를 차지함

```python
n = 65               # 정수 65
b = n.to_bytes(1)    # 바이트로 변환 -> b'A'
print(b)             # 컴퓨터가 실제로 저장하는 모습 (문자 A와 동일한 바이트)
```

- 파이썬에서는 여기에 더해, **모든 값이 '객체'** 라서 실제 데이터 앞에 **관리용 정보(헤더 16바이트)** 가 항상 붙음
  - 참조 카운트(8b) + 타입 정보 포인터(8b) = 최소 16b
- 그래서 파이썬의 숫자 하나가 C언어보다 훨씬 크게 나옴 → 뒤에서 계속 나오는 개념임

## 1단계: 숫자를 다루는 타입

### int (정수)

- 소수점 없는 수. **크기 제한이 없어서** 아무리 큰 수도 오류 없이 담김 (C언어와 다른 큰 장점)
- 값이 커질수록 메모리도 조금씩 늘어남

| 값 | 크기 | 설명 |
|---|---|---|
| `0`, `1` | 28b | 작은 정수 기본 |
| `2**64` | 36b | 값이 커지면 증가 |
| `2**1000` | 160b | 아주 큰 수 |

> 팁: 파이썬은 자주 쓰는 작은 정수(`-5 ~ 256`)를 미리 만들어 재사용함

### float (실수)

- 소수점이 있는 수. 내부적으로 64비트로 저장 → **값과 상관없이 항상 24b 고정**
- 주의: 컴퓨터는 소수를 완벽히 표현 못 해서 `0.1 + 0.2 != 0.3` 같은 미세 오차 발생
- 돈 계산처럼 정확도가 중요하면 `float` 대신 `decimal.Decimal` 사용

### bool (참/거짓)

- `True` / `False` 두 값만 가짐. 사실은 정수의 한 종류라 크기도 int와 같은 **28b** (`True == 1`)
- 조건 판단, 필터링 플래그에 사용

### complex (복소수)

- `3 + 4j` 형태의 복소수, **32b 고정**. 신호처리·주파수 분석(FFT) 등에서 사용

## 2단계: 여러 값을 묶는 타입

### 핵심 개념

- 리스트·튜플은 값을 직접 담지 않고, **값이 있는 곳을 가리키는 주소(포인터, 8b)** 만 나열해서 담음
- 그래서 `크기 = 껍데기 + 담긴 값들의 크기` 로 따로 계산해야 함

### list (리스트) — 수정 가능한 묶음

- 값을 자유롭게 추가·삭제·수정 가능. 빈 리스트 **56b**, 값 하나당 주소 **+8b**
- 순서가 있는 데이터 모음에 가장 많이 쓰임

```python
my_list = [1, 2, 3]      # 수정 가능한 목록
my_list.append(4)        # 뒤에 추가 가능
```

### tuple (튜플) — 수정 불가능한 묶음

- 한 번 만들면 못 바꿈. 그래서 리스트보다 **가볍고(빈 값 40b) 빠름**
- 좌표 `(x, y)`, 함수의 여러 반환값처럼 "고정된 한 세트"에 적합

### range — 반복 구간

- 시작·끝·간격 3개만 저장해서 **개수와 상관없이 항상 48b**
- `range(10억)` 도 메모리는 똑같음 → 반복문 인덱스에 쓰면 메모리 절약

---

## 3단계: 글자와 이진 데이터를 다루는 타입

### str (문자열) — 글자

- 텍스트를 담는 타입. 한 번 만들면 못 바꿈(불변)
- 담긴 글자 종류에 따라 글자당 크기가 자동으로 바뀜

| 문자열 | 크기 | 글자당 |
|---|---|---|
| `''` | 41b | 기본 오버헤드 |
| `'hello'` | 46b | 영문 1b |
| `'한글'` | 74b | 한글 2b |
| `'😀'` | 64b | 이모지 4b |

### bytes / bytearray — 순수 이진 데이터

- 이미지·오디오·네트워크 패킷처럼 **글자가 아닌 raw 데이터**를 다룰 때 사용
- `bytes` : 수정 불가(읽기용), 빈 값 33b
- `bytearray` : 수정 가능, 빈 값 56b
- `str`과 `bytes` 사이는 `encode()` / `decode()` 로 변환 (앞서 말한 "바이트 ↔ 해석" 개념과 연결됨)

---

## 4단계: 이름표로 찾는 타입

### dict (딕셔너리) — 키-값 사전

- `{"이름": "값"}` 처럼 **이름표(키)로 값을 즉시 찾음** (매우 빠른 조회)
- 3.7 버전부터 넣은 순서가 유지됨
- 빈 값 64b, 항목이 생기면 184b로 커짐 (빠른 조회를 위한 구조 비용)
- JSON, 설정값, 레코드 저장에 최적

### set / frozenset — 중복 없는 모음

- **중복을 자동 제거**하고, "이 값이 들어있나?"를 매우 빠르게 확인함. 순서는 없음
- 빈 값도 216b로 무거운 편 (빠른 검색을 위한 구조 때문)
- `set` : 수정 가능 / `frozenset` : 수정 불가(딕셔너리 키로도 사용 가능)
- 중복 제거, 교집합·합집합 같은 집합 연산에 사용

## 5단계: 날짜와 시간을 다루는 타입 (datetime)

- 파이썬 기본 제공 `datetime` 모듈로 날짜·시간을 정확히 계산할 수 있음
- 문자열 `"2026-07-05"` 로 저장하면 계산이 안 되므로, 시간 데이터는 반드시 전용 타입으로 다뤄야 함

| 타입 | 크기 | 담는 것 | 예시 |
|---|---|---|---|
| `date` | 32b | 날짜(년/월/일) | `2026-07-05` |
| `time` | 40b | 시각(시/분/초) | `13:30:00` |
| `datetime` | 48b | 날짜 + 시각 | `2026-07-05 13:30:00` |
| `timedelta` | 40b | 시간 간격/차이 | `3 days, 2:00:00` |

```python
from datetime import datetime, timedelta

now = datetime(2026, 7, 5, 13, 30)   # 특정 날짜+시각
tomorrow = now + timedelta(days=1)    # 시간 계산이 자유로움
diff = tomorrow - now                 # 두 시각의 차이 -> timedelta
print(diff)                           # 1 day, 0:00:00
```

- **대용량 시계열**에서는 개별 객체 대신 배열용 타입을 씀
  - `numpy.datetime64` : 요소당 **8b** 고정 (레이더 트랙 타임스탬프처럼 대량 시간값에 효율적)
  - `pandas.Timestamp` / `pd.to_datetime()` : 표(DataFrame)의 시간 컬럼 처리 표준

## 6단계: 대용량·표 데이터를 다루는 타입

순수 파이썬 타입은 대량 데이터에 비효율적임 → 데이터 분석/ML에서는 아래 타입을 씀.

### numpy 배열 — 대량 수치 데이터

- 리스트는 값마다 객체+주소라 무겁지만, numpy 배열은 **같은 타입 값만 연속으로** 저장해 오버헤드가 거의 없음

| 방식 | 정수 100만 개 | 비교 |
|---|---|---|
| python `list` | 약 36 MB | 무거움 |
| `np.int64` 배열 | 8 MB | 약 4.5배 절약 |
| `np.int32` 배열 | 4 MB | 약 9배 절약 |

- 자주 쓰는 dtype과 요소당 크기:

| dtype | 크기 | 용도 |
|---|---|---|
| `int8` / `uint8` | 1b | 이미지 픽셀, 마스크 |
| `int32` / `int64` | 4b / 8b | 일반 정수, 인덱스 |
| `float32` | 4b | **딥러닝/GPU 표준** |
| `float64` | 8b | 과학계산 기본(고정밀) |
| `bool_` | 1b | 필터 조건 마스크 |

> 팁: 딥러닝 데이터는 `float64` → **`float32`** 로 바꾸면 메모리 절반 + 속도 향상

### category (카테고리형) — 반복되는 라벨

- 같은 값이 계속 반복되는 문자열(예: 분류 라벨)을 **정수 코드 + 사전** 으로 바꿔 저장하는 pandas 전용 타입
- 원리: `"항공기"`, `"함정"` 같은 라벨을 매번 문자열로 저장하지 않고 `0`, `1` 번호로만 저장

| 저장 방식 | 라벨 10만 개 메모리 | 비교 |
|---|---|---|
| `object` (일반 문자열) | 약 7.0 MB | 문자열을 매번 저장 |
| `category` | 약 0.1 MB | **98.6% 절감** |

```python
import pandas as pd

# 분류 라벨처럼 값 종류가 적고 반복되는 컬럼에 효과적
labels = pd.Series(["항공기", "함정", "미상", "지상"] * 25000)
labels_cat = labels.astype("category")   # category 타입으로 변환 -> 메모리 급감
```

- **언제 쓰나**: 성별, 지역, 등급, **분류 라벨(track class 등)** 처럼 값의 종류가 적고 반복이 많은 컬럼
- 메모리 절감뿐 아니라 그룹 연산 속도도 빨라짐

### pandas DataFrame — 표 전체

- 행·열로 된 표 데이터를 다루는 타입. 컬럼마다 위 dtype(정수/실수/category/datetime64 등)을 지정해 메모리를 최적화함

## 실제 메모리 용량 한눈에 보기

| 타입 | 최소 크기 | 크기 규칙 |
|---|---|---|
| `NoneType` | 16b | 고정(값 없음) |
| `float` | 24b | 고정 |
| `bool` / `int` | 28b | int는 값 클수록 증가 |
| `complex` | 32b | 고정 |
| `date` | 32b | 고정 |
| `str` | 41b~ | 글자당 1/2/4b |
| `time` / `timedelta` | 40b | 고정 |
| `tuple` | 40b | 요소당 +8b |
| `datetime` | 48b | 고정 |
| `range` | 48b | 고정 |
| `list` | 56b | 요소당 +8b |
| `dict` | 64b | 항목 생기면 급증 |
| `set` / `frozenset` | 216b | 무거운 편 |

### 꼭 기억할 3가지

1. **모든 값에 16b 헤더가 붙는다** → 작은 값 수백만 개는 헤더만으로도 부담
2. **묶음 타입의 크기는 껍데기일 뿐** → 담긴 값까지 합쳐야 실제 총량
3. **대량 데이터는 numpy/category/datetime64** → 순수 파이썬 타입 대비 메모리 압도적 절약

## 데이터별 권장 타입 (선택 가이드)

| 다루는 데이터 | 권장 타입 | 이유 |
|---|---|---|
| 개수·인덱스·ID | `int` | 정확한 정수 |
| 측정값·확률 | `float` (ML은 `float32`) | 실수, ML 표준 |
| 금액·정밀 계산 | `decimal.Decimal` | 오차 방지 |
| 참/거짓 | `bool` | 의미 명확 |
| 텍스트 | `str` | 유니코드 지원 |
| 수정하는 목록 | `list` | 유연함 |
| 고정된 한 세트 | `tuple` | 가볍고 안전 |
| 이름표로 찾기 | `dict` | 빠른 조회 |
| 중복 제거·집합 연산 | `set` | 빠른 검색 |
| 이미지·패킷 | `bytes` / `bytearray` | raw 바이트 |
| **날짜·시간** | `datetime`, `datetime64` | 시간 계산 가능 |
| **반복되는 분류 라벨** | `category` | 메모리 대폭 절감 |
| 대량 수치 배열 | `numpy.ndarray` | 효율 압도적 |
| 표 형태 데이터 | `pandas.DataFrame` | 컬럼별 dtype 관리 |

## 정리 (치트시트)

- 컴퓨터는 모든 데이터를 **바이트**로만 저장하고, **데이터 타입은 그 바이트를 어떻게 읽을지 정하는 규칙**이다
- 파이썬은 모든 값이 객체라 **최소 16b 헤더**가 붙는다 → 그래서 숫자 하나도 28b씩 나온다
- **수정 가능**(list, dict, set, bytearray) vs **수정 불가**(int, float, str, tuple, date 등)의 구분이 동작과 성능을 가른다
- 날짜·시간은 문자열이 아니라 반드시 `datetime` 계열로 다뤄야 계산이 된다
- **반복되는 분류 라벨은 `category`** 로 바꾸면 메모리가 90% 이상 줄 수 있다
- 대량 데이터는 순수 파이썬 타입이 아니라 **numpy·pandas의 전용 dtype**으로 다뤄야 메모리와 속도를 모두 잡는다
