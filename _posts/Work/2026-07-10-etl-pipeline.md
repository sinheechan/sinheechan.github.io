| 항목 | 내용 |

| 사업 개요 | 군 운용 환경 레이더 수집되는 항적 데이터를 기반으로 항적 식별 분류 모델 구축 |
| 역할 | 데이터 정제·병합부터 모델 개발·검증까지 전체 데이터 파이프라인 설계·운영 및 사업 산출물 작성·관리 |

## 요구사항

- 대용량 라벨 데이터 납품 전 형식·값·무결성을 검증하는 객관적 품질검증 체계 구축
- 실제 작전 운용 시 데이터 처리부터 라벨링까지 실시간으로 수행하는 파이프라인 구성
- 개발된 AI 모델이 실전 환경의 항적 분류에 활용될 수 있는 수준의 분류 정확도 달성

## 문제 정의

- 작전일지·레이더 신호·신호 처리 결과가 상이한 기준으로 생성되어 공통 고유 ID 부재로 인한 데이터 통합의 어려움
- 대용량 라벨 데이터의 품질을 사람이 전수 검수하기 어려운 규모로 인한 품질 검증의 불확실성
- 새로운 운용 환경 추가 시 데이터 처리부터 라벨링까지 장시간 소요로 인한 운영 효율성 저하
- 실전 환경의 클래스 불균형으로 인한 소수 클래스 분류 정확도 확보의 어려움

## 문제 해결

- 데이터 통합 : 데이터 변수 분석을 통한 복합 키를 식별해 매칭 기준 확보 및 신뢰성 검증
- 휴먼에러 자동 보정: 상황 병 수기 기록 오류(자정 넘김·날짜 역전)를 앞뒤 데이터로 자동 감지·정정, 검수 시간 대폭 단축
- 자동 품질 검증: 산출 데이터 다항목 자동 점검, 검증 결과를 파이프라인 개선으로 환류
- 모델 개발·검증: 시계열 데이터를 동일 길이로 세그멘테이션하고 언더샘플링으로 클래스 균형을 맞춘 뒤, 다중 모델 비교·검증으로 평가 기준 분류 정확도 충족

## 기술 스택

`Python` `Tkinter` `scikit-learn` `PyTorch`
