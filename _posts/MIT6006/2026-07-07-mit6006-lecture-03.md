## 0. 세션 목표

- 강의에서 배운 개념을 실제 문제에 어떻게 적용하는지 익히는 것이 이 세션의 목표다.
- 정답 자체보다 "어떻게 접근하고, 왜 그렇게 접근하는가"가 핵심이다.
- 다루는 주제는 복잡도 감각 잡기, 블랙박스로 시퀀스 연산 구현하기, 덱(deque) 설계, 연결 리스트 뒤집기다.

## 1. 함수 증가율 순서 정렬

- 증가율 서열은 로그 < 다항식 < 지수 < 팩토리얼 순이다.
  - 로그는 입력이 커져도 거의 자라지 않고, 지수·팩토리얼은 입력이 조금만 커져도 폭발적으로 증가한다.
- 증가 속도가 같은 함수끼리는 `{ }`로 묶어서 표기한다 (예: `n`과 `n+√n`은 둘 다 Θ(n)).
- 공통 인수를 빼내 단순화하면 증가율 비교가 훨씬 쉬워진다 (예: `n·log n` vs `n·√n` → `log n` vs `√n` → 앞쪽이 느림).

| **입력 n** | **log₂n** | **n** | **n²**    | **2ⁿ** |
| -------- | --------- | ----- | --------- | ------ |
| **10**       | 약 3       | 10    | 100       | 1,024  |
| **100**      | 약 7       | 100   | 10,000    | 약 10³⁰ |
| **1,000**    | 약 10      | 1,000 | 1,000,000 | 천문학적   |

### 1-1. 정렬 규칙: 같은 속도는 묶는다

- 두 함수의 증가 속도가 같다면 하나의 그룹 `{ }`로 표기한다.

```text
Q. n, √n, n+√n 세 개를 정렬하면?
    - √n → 가장 느림 (맨 앞)
    - n과 n+√n → +√n은 n에 비해 무시할 만큼 작으므로 둘 다 Θ(n) → 같은 그룹
    - 결과: √n < {n, n+√n}
```

### 1-2. 실전 접근법: 공통 인수 빼내기

```text
Q. n·log n 과 n·√n 중 뭐가 빠를까?
    - 공통 인수 n을 빼면 → log n vs √n 비교만 남는다.
    - 증가율 서열에서 로그 < 다항식이므로 log n이 더 느리다.
    - 따라서 n·log n < n·√n.
```

## 2. 시퀀스 연산을 블랙박스로 구현

- 블랙박스 방식은 외부에 공개된 연산(`insert/delete_first/last`, 모두 O(1))만 사용해서 상위 기능을 조립하는 것이다. delete 연산은 삭제한 값을 반환한다.
- `swap_ends`: 양 끝 원소를 꺼내서 자리를 바꿔 다시 넣는다 → O(1).

![swap_ends: 양 끝을 꺼내 반대 자리에 다시 삽입](/assets/img/mit6006/mit6006-03-swap-ends.png){: .normal width="600"}

```python
def swap_ends(D):
    x1, x2 = D.delete_first(), D.delete_last()
    D.insert_first(x2); D.insert_last(x1)
```

- `shift_left(D, k)`: 맨 앞 원소 하나를 뒤로 옮기고, 나머지는 k-1짜리 문제로 재귀 호출한다 → O(k).

![shift_left: 맨 앞 원소를 하나씩 뒤로 옮기는 과정](/assets/img/mit6006/mit6006-03-shift-left.png){: .normal width="600"}

```python
def shift_left(D, k):
    if k < 1 or k > len(D) - 1: return
    D.insert_last(D.delete_first())
    shift_left(D, k - 1)
```

- 핵심은 재귀를 쓰면 상수 개의 정보와 불변식만 신경 쓰면 되기 때문에, 정확성을 논증하기가 쉬워진다는 점이다.

## 3. 동적 배열 → 양쪽 끝이 빠른 덱(Deque)

[각 자료구조의 한계]
- 


- 한계: 동적 배열은 앞쪽 삽입·삭제가 O(n)이고, 연결 리스트는 임의 위치 조회가 O(n)이다.
- 목표: 조회는 O(1), 양쪽 끝 삽입·삭제는 O(1) 상환(amortized)에 도달하는 것.
- 상환 O(1): 개별 연산이 가끔 비싸더라도 여러 번 반복했을 때 평균을 내면 O(1)이 되는 것 (예: 리스트의 `append`).
  - 성립 조건: 비싼 연산이 한 번 일어나기 전에 그만큼의 값싼 연산이 쌓여 있어야 한다.
- 접근 1: 동적 배열의 양쪽 끝에 여유 공간을 두고 확장하며, 축소할 때도 양쪽에 여유 공간을 남겨둔다.
- 접근 2: 동적 배열 2개를 등을 맞대듯 배치하고 한쪽은 역순으로 저장한다. 한쪽이 비면 남은 데이터를 반으로 나눠 재분배한다.

## 4. 연결 리스트 뒤집기 (Reorder Students)

- 할 일: 리스트 뒤쪽 n개 노드를 새 노드 없이 제자리에서 뒤집는다 (`next` 포인터만 조작).
- 직관: 손을 잡고 줄 서 있는 사람들이 앞사람 대신 뒷사람의 어깨로 손을 옮겨 잡는 것과 같다.
  - 손을 놓기 전에 원래 앞사람(`x_next`)을 먼저 기억해둬야 길을 잃지 않는다 — 이 부분이 가장 중요하다.
- 포인터 3개: `x`(나) / `x_prev`(뒷사람) / `x_next`(앞사람).
- 3단계로 진행한다: ① 뒤집을 구간의 경계 찾기 → ② 포인터 방향을 하나씩 뒤집기(반복) → ③ 양 끝을 다시 연결해 매듭 정리하기 (`a.next=c`, `b.next=None`).

## 5. 한 줄 정리 (알고리즘 관점)

- 1 : 복잡도 서열(로그 < 다항식 < 지수 < 팩토리얼)을 감으로 익히고, 공통 인수를 빼내 비교한다.
- 2 : 주어진 연산만으로 상위 기능을 조립하고, 재귀로 문제를 한 단계씩 줄여나간다.
- 3 : 두 자료구조의 장점을 합치되, 상환 분석으로 효율을 지킨다.
- 4 : 제자리 포인터 조작(3-포인터 뒤집기)으로 연결 리스트를 다룬다.

- 관통 메시지: 문제를 작은 단위로 쪼개고, 각 조각의 정확성과 비용을 따로 논증하라.
