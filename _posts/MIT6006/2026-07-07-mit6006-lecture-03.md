## 0. 세션 목표

- 강의에서 배운 개념을 실제 문제에 어떻게 적용하는지 익히는 것이 이 세션의 목표다.
- 정답 자체보다 "어떻게 접근하고, 왜 그렇게 접근하는가"가 핵심이다.
- 다루는 주제는 복잡도 감각 잡기, 블랙박스로 시퀀스 연산 구현하기, 덱(deque) 설계, 연결 리스트 뒤집기다.

[한 줄 정리]

- 1 : 복잡도 서열(로그 < 다항식 < 지수 < 팩토리얼)을 감으로 익히고, 공통 인수를 빼내 비교한다.
- 2 : 주어진 연산만으로 상위 기능을 조립하고, 재귀로 문제를 한 단계씩 줄여나간다.
- 3 : 두 자료구조의 장점을 합치되, 상환 분석으로 효율을 지킨다.
- 4 : 제자리 포인터 조작(3-포인터 뒤집기)으로 연결 리스트를 다룬다.
- 관통 메시지 : 문제를 작은 단위로 쪼개고, 각 조각의 정확성과 비용을 따로 논증하라.

## 1. 함수 증가율 순서 정렬

- 증가율 서열은 상수 < log n < 다항식(n, n²) < 지수(2ⁿ) < 팩토리얼(n!) 순으로 커진다.
- 두 함수가 같은 속도로 자랄 때는 하나의 그룹으로 묶어 표기한다.
- 복잡해 보이는 함수도 공통 인수를 빼내면 증가율을 쉽게 비교할 수 있다 (예: `n·log n` vs `n·√n` → `log n` vs `√n` → 앞쪽이 느림).

| 입력 n | log₂n | n | n² | 2ⁿ | n! |
|---|---|---|---|---|---|
| 10 | 약 3 | 10 | 100 | 1,024 | 3,628,800 (약 360만) |
| 100 | 약 7 | 100 | 10,000 | 약 10³⁰ | 약 10¹⁵⁸ |
| 1,000 | 약 10 | 1,000 | 1,000,000 | 천문학적 | 상상 불가 |

- 두 함수가 같은 속도면 하나의 그룹 `{ }`로 묶어 표기한다.

```text
Q. n, √n, n+√n 세 개를 정렬하면?
    - √n → 가장 느림 (맨 앞)
    - n과 n+√n → +√n은 n에 비해 무시할 만큼 작으므로 둘 다 Θ(n) → 같은 그룹
    - 결과: √n < {n, n+√n}
```

- 공통 인수 빼내기

```text
Q. n·log n 과 n·√n 중 뭐가 빠를까?
    - 공통 인수 n을 빼면 → log n vs √n 비교만 남는다.
    - 증가율 서열에서 로그 < 다항식이므로 log n이 더 느리다.
    - 따라서 n·log n < n·√n.
```

## 2. 시퀀스 연산을 블랙박스로 구현

- 블랙박스는 `insert`, `delete_first`, `delete_last` 등 외부 연산만 사용하며, 모두 O(1)이다. `delete`는 삭제한 값을 반환한다.
- `swap_ends`는 양 끝 값을 꺼낸 뒤 자리를 바꿔 다시 넣는 방식으로, O(1)에 동작한다.

![swap_ends: 양 끝 원소를 꺼내 반대 자리에 다시 삽입](/assets/img/mit6006/mit6006-03-swap-ends.png){: .normal width="600"}

```python
def swap_ends(D):
    x1, x2 = D.delete_first(), D.delete_last()
    D.insert_first(x2); D.insert_last(x1)
```

- `shift_left(D, k)`: 맨 앞 하나를 뒤로 옮기고, 나머지는 k-1 문제로 재귀 호출한다 → O(k).

![shift_left: 맨 앞 원소를 하나씩 뒤로 옮기는 과정](/assets/img/mit6006/mit6006-03-shift-left.png){: .normal width="700"}

```python
def shift_left(D, k):
    if k < 1 or k > len(D) - 1: return
    D.insert_last(D.delete_first())
    shift_left(D, k - 1)
```

- 핵심은 재귀를 쓰면 상수 개의 정보와 불변식만 신경 쓰면 되기 때문에, 정확성을 논증하기가 쉬워진다는 점이다.

## 3. 동적 배열 → 양쪽 끝이 빠른 덱(Deque)

- 우리가 원하는 건 다음 세 가지를 동시에 잘하는 자료구조다.
  - 중간 아무 데나 바로 읽기 (예: "5번째 항목 뭐야?")
  - 맨 앞에 넣고 빼기
  - 맨 뒤에 넣고 빼기
- 문제는 기존 자료구조들이 이 중 일부만 잘한다는 것이다.
  - 동적 배열은 앞쪽이 느리다 (앞에 끼우면 뒤를 다 밀어야 함).
  - 연결 리스트는 조회가 느리다 (몇 번째인지 세면서 따라가야 함).
- 우리가 원하는 건 결함이 하나도 없는 자료구조이며, 그게 바로 덱(deque)이다.
- 상환(amortized)이란, 가끔 한 번 비싼 작업이 있어도 그 전에 값싼 작업을 충분히 많이 했다면 평균적으로는 싸다는 개념이다.

### 3-1. 양쪽에 여유 공간 미리 만들기

- 동적 배열의 앞쪽 삽입이 느렸던 이유는 밀 공간이 없어서였다. 배열 양쪽에 미리 빈 공간을 만들어두면 앞이든 뒤든 밀지 않고 빈칸에 바로 넣을 수 있고, 이 방식으로 앞쪽 삽입도 O(1)이 된다.
- 빈칸을 다 쓰면 배열을 더 크게 재생성하면서 양쪽 여유를 다시 채운다. 이 재생성은 가끔 발생하는 비싼 작업이지만, 상환 분석으로 보면 평균은 여전히 저렴하다.
- 삭제로 배열이 텅 비어 크기를 줄일 때도 딱 맞게 줄이면 안 된다. 딱 맞게 줄이면 삽입·삭제가 반복될 때마다 늘렸다 줄였다를 반복하며 낭비가 생기므로, 줄일 때도 양쪽에 여유를 남긴다.
- 요약: 양쪽에 빈칸을 미리 만들어두고, 꽉 차면 상환 방식으로 다시 확장한다.

### 3-2. 리스트 두 개를 등 맞대기 (한쪽은 뒤집어서)

- 접근 1처럼 배열을 직접 구현하지 않고, 이미 있는 파이썬 리스트(동적 배열) 두 개를 조합해 덱을 만드는 방법이다.
- 파이썬 리스트는 끝(뒤)에서의 삽입·삭제(`append`/`pop`)는 빠르지만 앞에서는 느리다. 그래서 두 리스트 모두 "빠른 끝"만 쓰도록 구성한다.
  - `back` 리스트: 시퀀스 뒷부분을 그대로 저장한다. `back.pop()` → 시퀀스 맨 뒤 원소.
  - `front` 리스트: 시퀀스 앞부분을 뒤집어서 저장한다. 예를 들어 시퀀스가 `A B C D`면 `front`에는 `B A` 순으로 저장된다.
  - 이렇게 하면 시퀀스 맨 앞 원소가 리스트의 "끝"에 오게 되어 `front.pop()` → 시퀀스 맨 앞 원소가 된다.
- 양쪽 다 리스트의 빠른 끝(`pop`/`append`)만 쓰므로 앞/뒤 삽입·삭제 모두 O(1)이다.
- 문제는 한쪽 리스트가 완전히 비었을 때다. 예를 들어 `front`가 비었는데 앞에서 또 빼야 하면, `back`의 앞부분(느린 쪽)을 건드려야 하는 상황이 생긴다.
- 해결책은 `_rebalance`: 한쪽이 비면 남은 원소를 모아 양쪽에 반씩 다시 나눠 담는다.
- 이 재조정은 한 번에 O(n)이 걸리지만, 여기까지 오는 동안 값싼 연산을 충분히 쌓았으므로 상환 분석상 평균은 여전히 O(1)이다.
- 요약: 리스트 두 개를 등 맞대고 앞쪽은 뒤집어 저장해 양쪽 다 빠른 끝만 사용하며, 한쪽이 비면 반씩 재분배해 평균 O(1)을 유지한다.

## 4. 연결 리스트 뒤집기 (Reorder Students)

- 마지막 n개 노드를 제자리에서 뒤집는 문제다.
- 새 노드를 만들지 않고 `next`만 조작해 뒤쪽 n개 노드를 뒤집는다.
- 손을 잡고 줄 서 있는 사람들이 앞사람 대신 뒷사람의 어깨로 손을 옮겨 잡는 것과 같은데, 손을 놓기 전에 원래 앞사람(`x_next`)을 먼저 기억해둬야 길을 잃지 않는다.
- 포인터는 세 개다: `x`(나), `x_prev`(뒷사람), `x_next`(앞사람).
- 진행은 3단계다.
  - ① 뒤집을 구간 바로 앞의 고정 노드 `a`(경계)를 찾는다.
  - ② `x`, `x_prev`, `x_next`를 한 칸씩 옮기며 각 노드의 `next`를 반대로 돌리는 걸 반복한다.
  - ③ 매듭을 정리한다 — 경계 `a`의 `next`를 새 머리 `c`로 연결하고(`a.next=c`), 원래 구간 시작이던 `b`는 이제 꼬리이므로 `next`를 `None`으로 만든다(`b.next=None`).

![연결 리스트 뒤집기: x_prev/x/x_next 세 포인터로 next를 하나씩 반대로 돌리기](/assets/img/mit6006/mit6006-03-linked-list-reverse.png){: .normal width="700"}
