## 0. 개요

- 이 강좌의 핵심은 계산 문제를 해결하는 방법을 배우는 것
- 그리고 내가 찾은 해결 방법이 옳고 효율적이라는 것을 전달하는 것을 목적으로 둠

[세 가지 전달 요소]

- 옳음을 증명하는 것 : 이 해결 방법이 실제로 맞다는 것을 보이기
- 효율성을 논하는 것 : 이 해결 방법이 얼마나 자원(시간, 메모리 등)을 적게 쓰는지 설명하기
- 아이디어를 전달하는 것 : 내 사고 과정과 접근 방식을 남에게 이해시키기

## 1. 알고리즘의 정의

- 알고리즘 = 입력(`Input`)에 따라 출력(`Output`)이 나오는 과정이 옳다는 것을 증명하는 것
- 입력 개수는 정해져 있어도, 그 문제를 해결하는 방식(출력)은 사람마다 다를 수 있음
- 문제 해결 시 고정된 규칙(예: "이 교실 학생 중 생일 같은 사람 있는가?")이 아니라, 통상적으로 활용 가능한 일반화된 규칙을 정의하는 데 중점을 둠
- 좋은 알고리즘은 입력값이 임의의 크기를 가질 수 있고, 그 크기가 변해도 작동해야 함 (30명이든 10억 명이든)
- 즉, 입력값 크기가 일반적인(고정되지 않은) 문제에 대해 해결 가능한 것이 알고리즘
- 문제는 입력이 정의되어 있는 구체적 상황이고, 알고리즘은 입력값에 제한을 두지 않으며 입력을 주면 출력을 만들어내는, 크기가 고정된 절차/기계이다

## 2. 문제 해결 과정

문제 : 한 반의 생일이 일치하는 학생의 수 찾기

[증명의 기본 구조]

- 작은 값에 대해 일반화한 뒤, 귀납적 증명(induction)을 통해 더 큰 값에도 성립함을 보여야 함
- 핵심 명제 : "처음 K명의 학생 중 생일이 일치하는 학생이 있다면, 알고리즘은 K+1번째 학생을 인터뷰하기 전에 그 일치하는 쌍을 찾아낼 것이다"
- 이 명제가 귀납적 추론에 사용할 좋은 변수 K를 설정해줌

[증명 단계]

1. Base Case (기저 사례) : K = 0일 때부터 시작
2. 귀납적 가설 (Inductive Hypothesis) : 어떤 값 K에 대해 이 가설이 참이라고 가정 (즉, K명까지 살펴봤을 때 일치하는 쌍이 있으면 알고리즘이 반드시 찾아낸다고 가정)
3. 귀납적 단계 (Inductive Step) : K에서 K+1로 넘어갈 때, 두 가지 경우로 나눠서 생각함

| 경우 | 상황 | 결과 |
|---|---|---|
| 경우 1 | 이미 K명 안에서 일치하는 쌍이 있었던 경우 | 정답을 이미 찾은 상태 |
| 경우 2 | K명 안에서는 일치하는 쌍이 없던 경우 | K+1번째 학생까지 포함해서 비교 → 모든 학생을 하나씩 다 고려하는 무차별 대입(brute force) 방식으로 진행 |

[결론]

- 두 경우 모두 알고리즘이 정확하게 일치하는 쌍을 찾아낸다는 것이 보장되므로, 이 귀납적 증명을 통해 알고리즘의 정확성(correctness)을 증명할 수 있음

## 3. 알고리즘 효율성 측정

- 효율적인 알고리즘 = 단순히 빠른 것이 아니라, 같은 문제를 푸는 다른 방법들과 비교했을 때 얼마나 빠른지를 의미
- 컴퓨터로 직접 실행 시간을 재는 방식은 한계가 있음 : 컴퓨터 성능(하드웨어)에 따라 결과가 달라지기 때문
- 따라서 기계(하드웨어) 요인은 효율성 판단에서 제외함

- **알고리즘의 성능을 측정**할 때 **시간 대신 "연산 횟수"를 센다**
- 어떤 컴퓨터든 기본 연산 하나는 일정한 시간이 걸린다고 가정하고, 실제 시간 대신 기본 연산의 횟수를 셈
- 이 기준으로 알고리즘의 효율을 측정하는 방법이 점근 분석(asymptotic analysis)
- **핵심** : 시간을 재지 않고 입력 크기에 따라 연산 횟수가 어떻게 늘어나는지를 본다
- 성능은 입력에 따라 달라지므로, 알고리즘 성능은 입력값의 크기(n)를 기준으로 비교함 (입력이 배열, 그래프 단위가 되면 크기가 커지기 때문)

[점근 표기법]

| 표기 | 의미 |
|---|---|
| O (빅오) | 상한값 : "최악의 경우 이 정도 이하" |
| Ω (오메가) | 하한값 : "최소한 이 정도 이상" |
| Θ (세타) | 상한과 하한이 모두 일치할 때 : "정확히 이 정도" |

[효율적이라고 부르는 기준]

- 대표적인 성장 속도 : log n < n (선형) < n^c (다항, c는 상수)
- 이 수업에서 "효율적"이라고 말하는 일반적인 기준은 다항 시간(polynomial time) — log n, n, n² 등
- 반면 지수 시간(예: 2ⁿ)은 입력이 커질수록 처리 시간이 폭발적으로 늘어나 심각한 문제가 됨
- 로그 함수는 그래프상 거의 상수처럼 완만하게 증가하는 반면, 선형 함수는 대각선으로 계속 증가함

[계산 모델 : RAM 모델과 메모리 구조]

- 컴퓨터가 어떤 기본 연산을 쓰는지 모를 때는, "고정된 시간 안에 수행 가능한 작업"을 정의한 계산 모델을 세워야 하며, 그 기준이 컴퓨터의 RAM이다.
- RAM(Random Access Memory) = 메모리의 여러 위치에 일정한 시간 안에 임의 접근할 수 있다는 뜻
- 메모리는 본질적으로 비트(0과 1)의 연속체이며, CPU는 소량의 정보를 저장·변경·처리할 수 있고, 메모리의 임의 위치에서 데이터를 가져와 처리한 뒤 다시 돌려보내는 명령어를 가짐
- 다만 메모리의 모든 비트 하나하나에 주소가 있는 것은 아니고, 바이트(8비트) 단위로 주소가 지정됨
- CPU가 메모리에서 한 번에 가져와 처리할 수 있는 최대 덩어리(청크)의 크기 = 현대 컴퓨터 기준 64비트
- 즉 CPU가 데이터를 처리하는 흐름은 : 주소 지정 → 해당 덩어리(청크)를 CPU로 가져옴 → 처리 → 결과를 다시 메모리로 출력한다.

[자료구조와의 연결]

- 알고리즘의 성능은 결국 이런 기본 연산들을 기준으로 측정되므로, 그 기준을 제대로 세우려면 **자료구조에 대한 이해** 가 필수
- 자료구조의 핵심 : 대량의 데이터를 저장하고, 그 데이터에 대한 다양한 연산을 효율적으로 지원하는 것
- 정보의 양이 일정하지 않을 때(가변적일 때), 처리 속도를 높이기 위한 다양한 저장 방법을 다루는 것이 기초가 되어야 함
